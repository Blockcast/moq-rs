## 1. Context & decomposition

End goal: **A/V-synced playback in Shaka MSE from a real FFmpeg `moq_mmt` MMTP
multicast stream.** Too large for one spec → decomposed:

| Sub-project | Scope | Repos | Depends on |
|---|---|---|---|
| **A — Video playback (THIS spec)** | reassembled video MFU → transmux → MSE; frames render | shaka-player (+ a parser-FEC slice, see §4.6) | — (unblocked) |
| **B — Publisher audio emission** | FFmpeg `moq_mmt` audio MMTP track; `moq-pub-mmtp` publish; catalog | FFmpeg fork, moq-rs | — |

### Decision record (brainstorm + eng-review 2026-05-30)
- **Scope of A:** full video playback, provable in the live/Karma MSE harness. Audio/sync → B/C.
- **D-arch1 — transmux: dedicated `MmtpTransmuxer`, PINNED thin-delegating.** Calls the shared `H264` + `Mp4Generator` utils directly; only MMTP-specific glue; MUST NOT copy LocTransmuxer's ~615-line body. Extract any LOC orchestration worth sharing into a helper first (Beck: make-the-change-easy).
- **D-cq1 — timing from the carried MMTP timestamp.** `startTime` = per-MFU MMTP `timestamp` (NTP-short → seconds), **normalized** so the first accepted timestamp maps to the live-edge offset (Codex #9). NOT an accumulated frame counter. `duration` from catalog `frameRate`. `moof` `baseMediaDecodeTime` in the `moov`'s timescale.
- **D-init — stripped catalog `moov` is the PRIMARY init; transmuxer is MEDIA-ONLY.** Strip the `mmpu` box from catalog `track.initData` → clean `ftyp+moov`, emitted upfront. `MmtpTransmuxer` synthesizes only `moof+mdat`, matched to that `moov`'s timescale + track_id. No SPS/PPS init synthesis; no catalog-init "fallback" (no silent fallbacks).
- **D-cq2 — FEC parser trailer fix is IN SCOPE for A (absorbs the T1.FEC parser-parity slice).** `mmtp_parser.js` currently models `source_FEC_payload_ID` as a **prefix**; per ISO/IEC 23008-1:2023 §C.5.2 it is a **trailer** past MPU `payload_length` (confirmed: `mmt_core/header.rs:255-259`, `moqenc_mmt.c:103,4453`). Fix is cross-repo: JS parser + Rust `mmt_core` vector generator + regenerate the parity vector + regression test. (FEC is OFF in the current test stream — this closes the latent gap, not a live bug.)
- **D-mime — prove the MIME path (Codex #4).** Verify `TransmuxerEngine` accepts `moq/mmt; codecs="avc1…"` and emits `video/mp4; codecs="avc1…"` to MSE (register MIME + `convertCodecs`/`isSupported` like LocTransmuxer) BEFORE SourceBuffer creation.
- **D-seq — segment-index sequencing (Codex #5/#8).** Init emitted upfront; **media appending gates on the first RAP frame** (`rapFlag=1`; test agreement vs NAL type 5). `processMmtpTrack_` must not publish a `SegmentReference` before init + first-RAP exist.
- **D-perf1 — explicit live eviction.** `segmentIndex.mergeAndEvict` + zero-DVR `setSegmentAvailabilityDuration`, mirroring the LOC branch — bound memory over multi-hour streams.
- **D-seg — per-frame media segments.** One `SegmentReference` per MFU (matches LOC, lowest latency). Volume bounded by D-perf1.
- **D-bframe — no composition offsets for A (Codex #10).** Synthesized `moof` emits no `ctts`/composition offset; PTS=DTS invariant documented (test stream has no B-frames).
- **D9 — SUPERSEDES D-init: init is synthesized from in-band SPS/PPS (LocTransmuxer pattern), not the catalog `moov`.** Plan-grounding (`getAvcStreamInfo_` + the `transmux()`→`{data,init}` contract) showed catalog-moov-primary adds new `avcC`-parsing for no real benefit (the transmuxer always produces init from a codec config; the RAP-frame wait is unavoidable regardless). `MmtpTransmuxer` caches `videoConfig` from the first IDR's in-band SPS/PPS and `Mp4Generator` emits `{init, media}`. **The `mmpu`-strip task is DROPPED.** No catalog init, no fallback (resolves Codex #6 cleanly). See `.planning/m4-t1.2-video-playback-plan.md`.

## 2. Ground truth (T1.7; memory `ffmpeg-moq-mmt-multicast-wire-format.md`)

- Multicast leg = **Init (FT=0)** `MMTP+MPU+(ftyp+mmpu+moov)` + **MFU (FT=2)** only. FT=1/moof never on wire.
- **MFU payload = AVCC length-prefixed NAL** (H.264; SEI `nal_type=6` then coded NALs). Media must be synthesized.
- Init bytes == catalog `track.initData`; Shaka reads it at `msf_parser.js:685`. Layout `ftyp+mmpu+moov` → strip `mmpu` for MSE (D-init).
- **Timing:** MMTP `timestamp` = `us_to_ntp_short(base_pts + i·sample_dur)`, constant across a frame's fragments, monotonic per frame, on every packet (the per-frame PTS source).
- Test stream = testsrc2 15fps GOP15, **no B-frames → PTS=DTS**. `mpu_sequence` = group_id (GOP).
- **FEC `source_FEC_payload_ID` is a TRAILER** past `payload_length` (ISO 23008-1 §C.5.2); FEC OFF in the test stream.

## 3. Existing wiring mirrored (shaka-player)

- `processTrack_` (~681) branches `isLoC`/`isMmtp` on `packaging` + `selectionParams.codec`.
- LOC branch is the template: `createSegmentIndex` → `subscribeToTrack(obj)` → `{startTime,duration,payload}` → `new SegmentReference` + `setSegmentData` → `mergeAndEvict` → `notifySegments` (msf_parser.js:880-954). NAL→fMP4 runs in `MediaSourceEngine`/`TransmuxerEngine` at append via a registered `shaka.extern.Transmuxer` (LocTransmuxer).
- Current MMTP path = observe-only (`observeMmtpTrack_` → log). `MmtpTrackProcessor`/`MfuReassembler` emit `'mfu'` records (complete frame NAL + group/subgroup/rapFlag/timestamp) — **reused unchanged**.

## 4. Design

### 4.1 Components
- **`shaka.transmuxer.MmtpTransmuxer`** (new, `lib/transmuxer/mmtp_transmuxer.js`) — a `shaka.extern.Transmuxer` registered in `TransmuxerEngine`, keyed `moq/mmt`, converting to `video/mp4` (D-mime). **Media-only**: takes a reassembled MFU (AVCC NAL) + timing (via the `SegmentReference` at `transform()`), synthesizes `moof+mdat` via shared `H264`+`Mp4Generator`, matched to the active `moov`'s timescale/track_id. **Thin-delegating** (D-arch1): no LocTransmuxer body copy. No `ctts` (D-bframe).
- **mmpu-strip helper** — strips the `mmpu` box from catalog `initData` → `ftyp+moov`; the `moov` is parsed for timescale + track_id and emitted as the MSE init upfront (D-init). Unit-tested.
- **`processMmtpTrack_`** in `msf_parser.js` (replaces `observeMmtpTrack_`) — `isMmtp` branch parallel to `isLoC`: builds the `Stream` (`moq/mmt` mime), emits the stripped init upfront, and in `subscribeToTrack` feeds objects to the reassembler, emitting one `SegmentReference` per completed MFU once the first RAP frame is seen (D-seq, D-seg), with eviction (D-perf1).
- **`mmtp_parser.js`** (modify) — read `source_FEC_payload_ID` as a trailer (D-cq2).
- **Reused unchanged:** `MmtpTrackProcessor`/`MfuReassembler`, `TransmuxerEngine`, `MediaSourceEngine`, `SegmentReference`/`InitSegmentReference`, `presentationTimeline`.

### 4.2 Data flow
catalog parse → strip `mmpu` → emit `ftyp+moov` init upfront (timescale/track_id cached).
`subscribeToTrack(obj)` → `MmtpTrackProcessor.process`:
- `'mfu'`: if no RAP seen yet and this frame's `rapFlag=0` → drop (mid-GOP join). On first `rapFlag=1` and after: `startTime` = normalized MMTP timestamp, `duration` = catalog frameRate (D-cq1) → `SegmentReference` + `setSegmentData(nalFrame)` (`moq/mmt`) → `mergeAndEvict` → `notifySegments`. At append, `TransmuxerEngine`→`MmtpTransmuxer`→`video/mp4` moof+mdat → SourceBuffer.
- `'init'` record → ignored (init from catalog).
- `'repair'`/`'non-media'` → dropped (T1.FEC policy).
- incomplete fragment → `null` (bounded by reassembler cap).
- `FragmentType.FRAGMENT` → **throw**.

### 4.3 Timing & init
- **Init:** stripped catalog `moov` upfront (D-init). Available before any media → no init-sequencing race.
- **Media start:** first frame with `rapFlag=1` (D-seq); earlier mid-GOP frames dropped.
- **startTime:** MMTP `timestamp` decoded (NTP-short → s), normalized to first-accepted → live-edge offset (D-cq1, Codex #9).
- **duration:** catalog `frameRate`. `moof` `baseMediaDecodeTime` in `moov` timescale.
- **No `ctts`** (D-bframe).

### 4.4 MIME registration (D-mime)
`MmtpTransmuxer.isSupported('moq/mmt; codecs="avc1…"')` true; `convertCodecs` → `video/mp4; codecs="avc1…"`; verify `MediaSourceEngine` builds the SourceBuffer from the converted MIME. Covered by a unit test + the render E2E.

### 4.5 Error handling (error-on-bad-init; no silent defaults/fallbacks)
- Missing catalog `frameRate`/`timescale`, or `initData` that fails `mmpu`-strip into a valid `ftyp+moov` → **throw**.
- `FragmentType.FRAGMENT` → **throw**.
- Reassembly bounded by the required `maxBufferedMfus` cap (no timers).
- No catalog-init fallback (D-init).

### 4.6 FEC parser-parity fix (in scope — D-cq2)
`mmtp_parser.js`: when `fec_type==1`, read `source_FEC_payload_ID` (4B) as a **trailer** after the MPU payload (`payload_length`-bounded), not a prefix. Reconcile the Rust `mmt_core` vector generator + regenerate `test/test/assets/mmtp_packet_vectors.json`; the existing `fec_type==1` vector + `mmtp_parser_unit.js:43` get a **regression** assertion on payload-boundary correctness. Repair packets (`fec_type==2`, front-id) unchanged — still T1.FEC.

### 4.7 Testing (all gaps + 3 E2E — D-test1)
- **Unit:** `MmtpTransmuxer` (media moof+mdat valid, matches moov params); mmpu-strip; MMTP-timestamp decode (monotonic + normalization); `processMmtpTrack_` throws (FRAGMENT, missing timing); `'init'` ignored; **FEC trailer regression** (CRITICAL); MIME `isSupported`/`convertCodecs`.
- **Integration (Karma):** extend the observe harness (`d88d3a3c2`) to append to MSE + assert `buffered.length>0`, `currentTime` advances.
- **E2E (extend `.planning/m4-t1.7-e2e.sh`):** (1) render — frames play; (2) mid-stream join — start mid-GOP, first RAP starts playback; (3) loss/reorder — drop/shuffle packets, assert timestamp-derived timeline holds (CQ#1's failure mode).

## 8. Parallelization

- **Lane A (shaka-player):** `MmtpTransmuxer` + mmpu-strip + `processMmtpTrack_` + JS FEC-parser fix + tests. Sequential (shared `lib/msf` + `lib/transmuxer`).

Lane B feeds Lane A's FEC regression test (the vector). Run B first or in parallel; converge at the FEC regression test. Conflict risk low (different repos).
